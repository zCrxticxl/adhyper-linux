# Security policy

## Reporting a vulnerability

Do not report vulnerabilities publicly. Use the maintainer contact route listed in the repository profile and include affected distribution, script version, reproduction steps, and impact.

We aim to acknowledge reports within 7 days. Please do not attach passwords, private paths, or full system logs.

## Safety boundary

Changes must remain opt-in, logged, backed up, and revertible. Risky operations must be documented and excluded from unattended execution unless the user explicitly chooses them.
